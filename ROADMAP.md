# Roadmap

## Улучшения

### Режим «Табу» в Шляпе
Добавить режим, в котором при объяснении слова отображается список запрещённых слов. Использование табу-слов — нарушение. Усложнённая версия классической Шляпы.

### Рисование в Крокодиле: инструменты
- Выбор толщины линии (тонкая / средняя / толстая)
- Ластик (рисует белым или стирает по области)
- Заливка области цветом

### Рисование в Крокодиле: цвета
Палитра из 6-8 базовых цветов (чёрный, красный, синий, зелёный, жёлтый, коричневый, оранжевый).

### Рисование в Крокодиле: UX
- Таймер обратного отсчёта overlay на canvas
- Сглаживание штрихов у зрителей
- Адаптация canvas под мобильные экраны
- Горячие клавиши: Ctrl+Z для undo

### Рисование в Крокодиле: геймплей
- Чат для угадывающих — ввод слов, автоматическая проверка сервером (вместо ручного markCorrect)
- Подсказки: через N секунд показать первую букву, через 2N — количество букв
- Очки за скорость: чем быстрее угадал — тем больше очков (и рисующему тоже)
- Все угадывают (не только первый) — разное количество очков за порядок угадывания

### Коммуникация между игроками
Fishka должна быть доступна для всех сценариев использования:

**Сценарий 1: Офлайн (рядом друг с другом)**
Игроки в одной комнате, телефон — интерфейс, общение голосом. Чат не нужен, всё работает как есть.

**Сценарий 2: Онлайн + внешний созвон**
Игроки удалённо, общаются через Discord/Zoom/Telegram. Игра — только игровой интерфейс, голос — снаружи.

**Сценарий 2a: Встроенная аудио-конференция**
Добавить voice chat прямо в Fishka (как в Discord). Убирает необходимость во внешнем созвоне. WebRTC peer-to-peer или SFU-сервер.

**Сценарий 3: Онлайн без голоса**
Игроки удалённо, без возможности созвона. Необходимы:
- Текстовый ввод ответов в Шляпе и Крокодиле (автоматическая проверка сервером)
- Возможность писать варианты слов вместо устного угадывания

**Сценарий 3a: Эфемерный чат (TikTok-стиль)**
Глобальный текстовый чат для всех игр и лобби. Сообщения исчезают через несколько секунд после отправки. Лёгкий способ общения без созвона. Не замусоривает экран — пропадает автоматически.

Архитектурно нужно продумать:
- Единый chat transport (WS message relay, как drawStroke) — без persist
- Текстовый ввод ответов как отдельный game action с серверной проверкой
- Voice chat: WebRTC signaling через существующий WS, STUN/TURN серверы
- UI: плавающие сообщения поверх игрового экрана (overlay)

## В планах

### Мафия (mafia)
Ролевая игра с тайными ролями. Днём город голосует за подозреваемого, ночью мафия выбирает жертву.

### Викторина (trivia)
Вопросы с вариантами ответов. Кто быстрее и больше — тот побеждает. Категории, таймер на ответ, таблица лидеров.

### Codenames (codenames)
Командная игра. Поле 5×5 со словами. Капитан даёт подсказку (слово + число), команда выбирает карточки. Нельзя попасть на карточку соперника или убийцу.

### Виселица (hangman)
Один загадывает слово, остальные угадывают по буквам. Ограниченное количество попыток. Классика.

### Морской бой (battleship)
Игра 1 на 1. Каждый расставляет корабли на поле 10×10, затем по очереди стреляют. Скрытое состояние — каждый видит только своё поле и выстрелы по сопернику.

### Кто я? (who-am-i)
Каждому игроку назначается персонаж, которого видят все кроме него. Игрок задаёт вопросы да/нет, остальные голосуют. Цель — угадать своего персонажа.

### Свояк (jeopardy)
Поле категорий с вопросами разной стоимости. Игроки выбирают вопрос, кто быстрее нажал — тот отвечает. Правильный ответ — очки, неправильный — штраф.
